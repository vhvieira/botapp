FreeCBR

This is a GUI application, a command line utility, a Java bean, a native ActiveX object, a web application and a general Java  API for Case Based Reasoning.

The current homepage is http://freecbr.sourceforge.net/

GUI application

In MS Windows environment, if Java runtime is installed and set up correctly, it is possible to run FreeCBR.exe.

Otherwise, to run the package with a graphical user interface, please run "java -jar {path to the file}FreeCBR.jar" or "java -cp {path to the file}FreeCBR.jar FreeCBR.GUI". If the environment is set up correctly it is also possible to start the application by clicking here.

Command line utility

To run the package as a command line utility, please run "java -cp {path to the file}FreeCBR.jar FreeCBR.Console".

Java bean

To use the package for example in a JSP-solution, the suggested method is to use
<jsp:useBean id="CBRBean" scope="application" class="FreeCBR.CBR"/>
<%
  CBRBean.setDatafile("xxxx");
  CBRBean.readData();
%>
 and in the result page use
<jsp:useBean id="CBRBean" scope="application" class="FreeCBR.CBR"/>
<%
  CBRBean.search(request);
%>
 See also the example jsp pages in the FreeCBR.war file, more information in the Web application section.

ActiveX object

ActiveX is now supported natively, no need for any Java at all. To install the ActiveX component, just run ActiveX/install.bat.

To use the component, run for example
  dim obj
  set obj=createobject("FreeCBR.CBR")
  obj.setINFINITY_CONSTANT 3
  set obj = nothing

There is a small test script at ActiveX/test.vbs.

There is also an example Excel file that very briefly shows how FreeCBR can be integrated in a VBA environment.

Please note that the "normal" Java API (as described in the API documentation) can be used when using the ActiveX component, but it might be easier to use:

  • getFeatureValueAX instead of getFeatureValue
  • getUsedStringValuesAX instead of getUsedStringValues
  • searchAX instead of search

Web application

There are three .jsp pages within the war-package. They are named "test.jsp", "page1.jsp" and "page2.jsp". test.jsp displays information about the case set, page1.jsp allows searches in the case set and test2.jsp will display the result of the search. The easiest way to test the web application is to put FreeCBR.war in the JSP root directory, wait for it to be unpacked, and then open http://localhost/FreeCBR/index.html.

Note that if another port is used it must be specified. If for example port 8080 is used by the web server the URL:s should look like http://localhost:8080/FreeCBR/index.html

If no JSP engines are available, I would recommend to install for example the freely available Apache Tomcat. If that is not an option, it is possible to open the FreeCBR.war using for example Winzip or another zip compatible application, perhaps by first renaming the FreeCBR.war file to FreeCBR.zip.
